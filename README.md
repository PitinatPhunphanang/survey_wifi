# Wi-Fi Survey Pro Dashboard

ระบบ Web Dashboard สำเร็จรูปสำหรับวิเคราะห์และทำรายงานผลการสำรวจสัญญาณ Wi-Fi (Wi-Fi Site Survey) ถูกออกแบบมาให้ทำงานร่วมกับ Script Python ที่ใช้ `iperf3` ในการเก็บข้อมูล 

ระบบนี้สร้างด้วย **Next.js 16 (App Router)**, **Tailwind CSS** สำหรับความสวยงาม, และ **Recharts** สำหรับการสร้างกราฟขั้นสูง

---

## 📂 โครงสร้างโปรเจกต์ (Project Structure)

โปรเจกต์นี้ใช้โครงสร้างของ Next.js แบบ App Router (`src/app`) โดยแบ่งหน้าและคอมโพเนนต์หลักๆ ดังนี้:

### 1. หน้าจอหลัก (Pages)
- **`src/app/page.tsx` (Dashboard)**
  - หน้าจอหลักสำหรับ **"ดูหรือกรอกข้อมูลทีละ 1 จุด"** (Single Point Analysis)
  - เมื่อนำไฟล์ Excel มา Import ในหน้านี้ ระบบจะดึงข้อมูล **บรรทัดสุดท้าย (จุดล่าสุดที่เพิ่งเทสเสร็จ)** มาแสดงบนเกจวัดและกราฟ เพื่อให้วิศวกรหน้างานตรวจสอบค่าได้ทันที
  
- **`src/app/report/page.tsx` (Floor Report)**
  - หน้ารายงานสรุปแบบ **"ภาพรวมทั้งชั้น/ตึก"** (Aggregated Floor Analytics)
  - เมื่อ Import ไฟล์ Excel ระบบจะอ่านทุกบรรทัด และ **จัดกลุ่มตามคอลัมน์ Floor อัตโนมัติ**
  - มีกราฟแท่ง (Bar Charts), สัดส่วนคลื่นความถี่ (Donut Chart), ตารางจุดที่แย่ที่สุด (Worst Points) และ **ระบบวิเคราะห์ข้อความสรุปอัตโนมัติ (Auto Summary)** เหมาะสำหรับแคปหน้าจอลงเล่มรายงาน

- **`src/app/reference/page.tsx` (Reference Guide)**
  - หน้าคู่มืออ้างอิงเกณฑ์การวัดผล (Thresholds) อธิบายว่าค่า Good, Fair, Poor ของสัญญาณและ Throughput วัดจากอะไร

### 2. คอมโพเนนต์ (Components) - อยู่ใน `src/components/dashboard/`
- **`VisualCharts.tsx`**: ตัวจัดการกราฟทั้งหมดบนหน้า Dashboard 
- **`MetricCard.tsx`**: ตัวการ์ดแสดงผลตัวเลขและเกจสี (Good/Fair/Poor)
- **`ReportSummary.tsx`**: แผงสรุปผลรวม (Overall Health) ว่าผ่านหรือไม่ผ่าน

### 3. ฟังก์ชันและระบบหลังบ้าน (Libraries) - อยู่ใน `src/lib/`
- **`evaluation.ts`**: หัวใจหลักของการให้คะแนน! ไฟล์นี้รวม Logic (If-Else) ที่คอยคำนวณว่าค่า Ping, RSSI, Jitter ที่ได้มา ตกอยู่ในเกณฑ์สีเขียว (Good), เหลือง (Fair), หรือ แดง (Poor)
- **`storage.ts`**: ระบบบันทึกข้อมูลลง LocalStorage เพื่อให้เว็บทำงานแบบ Offline-first ได้
- **`export.ts`**: ฟังก์ชันสำหรับ Export ข้อมูลเป็น CSV, JSON และระบบ Print หน้าจอ

### 4. การตั้งค่าสไตล์ (Styling)
- **`src/app/globals.css`**: เก็บสีธีมหลัก โดยตั้งค่าเป็น Dark Mode สไตล์ "Soft Slate" (สีเทาเข้มอมน้ำเงิน) เพื่อให้อ่านง่าย ถนอมสายตา และดูมีความเป็น Enterprise

## ⚙️ การเชื่อมต่อกับ Python Script ของคุณ

ระบบนี้ไม่ได้ต่อ Database ใดๆ แต่ใช้การอ่านไฟล์ Excel (`.xlsx`) ที่ได้จากโปรแกรม Python ของคุณโดยตรง (ประมวลผลบนเบราว์เซอร์ ไม่มีการอัปโหลดขึ้นเซิร์ฟเวอร์ ปลอดภัย 100%)

**หลักการทำงาน:**
1. ในหน้าเว็บมีการเรียกใช้งานไลบรารีชื่อ `xlsx`
2. เมื่อคุณกดอัปโหลดไฟล์ ฟังก์ชัน `handleImportExcel` จะทำการแปลง Sheet เป็นข้อมูล JSON
3. ระบบจะทำการ **Map ชื่อคอลัมน์** (เช่น `RSSI_dBm`, `TCP_Download_Mbps`) ให้ตรงกับตัวแปรที่ใช้ใน TypeScript (`src/types/index.ts`)
4. ถ้าระบบหาคอลัมน์เจอ กราฟและตัวเลขทั้งหมดจะเด้งขึ้นมาแสดงผลแบบ Real-time ทันที

## 🚀 ฟีเจอร์เด่นสำหรับทำรายงาน (Report Features)

- **High-Resolution Charts:** กราฟทุกตัว (Recharts) ถูกเขียนโค้ดปรับความสูง (Height) ให้กว้างขึ้น และเพิ่ม `LabelList` เพื่อให้ตัวเลขลอยอยู่บนแท่งกราฟเสมอ ทำให้เวลา Capture รูปไปแปะในเอกสาร มีความคมชัดระดับสูงโดยไม่ต้องเอาเมาส์ไปชี้
- **Reference Lines:** มีเส้นประบอกเกณฑ์มาตรฐานแทรกอยู่ในทุกกราฟ (เช่น เส้นแดงที่ -75 dBm) 
- **Auto Summary Logic:** ระบบวิเคราะห์ค่าเฉลี่ยของชั้นนั้นๆ และเขียนเป็น "ข้อความภาษาไทย" ให้อัตโนมัติ เพื่อให้คุณนำไปก๊อปปี้วางในบทสรุปของเล่มรายงานได้เลย โดยไม่ต้องนั่งแปลกราฟเอง
- **Critical Points Analytics:** ระบบจะควานหาจุดที่แย่ที่สุดในแต่ละด้าน (Worst RSSI, Max Ping, Max Jitter) และสกัดออกมาเป็นตาราง Action List เพื่อให้ง่ายต่อการวางแผนแก้ไขจุดบอด
